¿Qué es y para qué sirve Git?

Git es un software de control de versiones diseñado por Linus Torvalds. Fue diseñado para que el mantenimiento de versiones de aplicaciones sea eficiente, rápido y confiable.

Su propósito es llevar registro de los cambios en archivos y coordinar el trabajo que varias personas pueden hacer sobre esos archivos. Es especialmente útil para proyectos de desarrollo de software, pero también se puede utilizar para cualquier tipo de archivo. Git permite a los desarrolladores colaborar, mantener un historial de cambios y revertir a versiones anteriores si es necesario.

Desarrollo colaborativo

El desarrollo colaborativo es un modelo de desarrollo basado en la disponibilidad pública del código y la comunicación vía internet. Este modelo se hizo conocido gracias a su utilización para el desarrollo del kernel de Linux en 1991, pero se ha extendido a una amplia variedad de proyectos de software y otros tipos de proyectos creativos.

Podríamos decir que en el desarrollo colaborativo Git proporciona una plataforma para que un gran número de desarrolladores puedan trabajar en conjunto de una manera más fácil, menos propensa a errores y rápida de implementar. Git permite a los desarrolladores compartir su código, colaborar en proyectos y mantener un historial de cambios, lo que facilita la gestión de proyectos de desarrollo de software a gran escala.

¡Descargá Git!

Para descargar Git, podés ir al sitio web oficial de Git en git-scm.com y hacer click en el botón de Install en naranja, donde recomendará la versión adecuada para tu sistema operativo.

Comandos

Configuración

git --version
Este comando se utiliza para verificar la versión de Git instalada en tu sistema. Al ejecutarlo, Git mostrará la versión actual que tienes instalada, lo que es útil para asegurarte de que estás utilizando una versión compatible con tus necesidades o para verificar que la instalación se realizó correctamente. 

git config --global --list
Este comando se utiliza para mostrar la configuración global de Git. Al ejecutarlo, Git mostrará una lista de todas las configuraciones globales que has establecido, incluyendo tu nombre de usuario, correo electrónico y otras opciones de configuración. 

git config --global user.name "Tu Nombre"
Este comando se utiliza para configurar el nombre de usuario global en Git. Al ejecutarlo, Git asignará el nombre que especifiques a todas las confirmaciones que realices en tu sistema. 

git config --global user.email "tu.email@example.com"
Este comando se utiliza para configurar el correo electrónico global en Git. Al ejecutarlo, Git asignará el correo electrónico que especifiques a todas las confirmaciones que realices en tu sistema. 

git config --global user.password "tuContraseña"
Este comando se utiliza para configurar la contraseña global en Git. Sin embargo, es importante destacar que Git no recomienda almacenar contraseñas en texto plano utilizando este comando, ya que puede representar un riesgo de seguridad. En su lugar, se recomienda utilizar métodos de autenticación más seguros, como el uso de claves SSH o tokens de acceso personal para autenticarte con servicios de alojamiento de código como GitHub o GitLab. 

git config --global core.autocrlf true
Este comando se utiliza para configurar la conversión automática de finales de línea en Git. Al ejecutarlo, Git convertirá automáticamente los finales de línea a un formato específico según el sistema operativo que estés utilizando.

  • true: para windows
  • false: para Linux/Mac

Esto es especialmente útil cuando trabajas en proyectos que involucran a desarrolladores que utilizan diferentes sistemas operativos, ya que ayuda a evitar problemas de compatibilidad relacionados con los finales de línea. 

git config --global init.defaultBranch main
Este comando se utiliza para configurar el nombre de la rama predeterminada en Git. Al ejecutarlo, Git establecerá el nombre de la rama principal como "main" en lugar de "master", que era el nombre predeterminado en versiones anteriores de Git.

Control de versiones

git init
Este comando se utiliza para inicializar un nuevo repositorio de Git en el directorio actual. Al ejecutarlo, Git creará un nuevo repositorio vacío en ese directorio, lo que te permitirá comenzar a rastrear los cambios en tus archivos y realizar confirmaciones. 

git add archivo
Este comando se utiliza para agregar un archivo al área de preparación de Git. Al ejecutarlo, Git incluirá el archivo especificado en la próxima confirmación. También se puede utilizar el comando: 
git add .
para agregar todos los archivos modificados en el directorio actual al área de preparación. 

git commit -m "Mensaje de confirmación"
Este comando se utiliza para crear una nueva confirmación en Git. Al ejecutarlo, Git tomará los archivos que has agregado al área de preparación y los guardará como una nueva versión en el historial del repositorio. El mensaje de confirmación que proporcionas con la opción -m debe describir brevemente los cambios que has realizado en esa confirmación para que otros desarrolladores puedan entender fácilmente el propósito de esa versión. 

git merge nombreRama
Este comando se utiliza para fusionar los cambios de una rama con la rama actual. Al ejecutarlo, Git integrará los commits de la rama especificada en la rama en la que te encuentras. 

git push
Este comando se utiliza para enviar los cambios locales al repositorio remoto. Al ejecutarlo, Git subirá tus commits al repositorio remoto, lo que permitirá que otros desarrolladores vean y colaboren en tus cambios. Se recomienda ejecutar este comando especificando la rama a la que deseas enviar los cambios y el repositorio remoto, por ejemplo: 
git push origin main
. 

git pull
Este comando se utiliza para obtener los cambios del repositorio remoto y fusionarlos con la rama actual. Al ejecutarlo, Git descargará los commits más recientes del repositorio remoto y los integrará en tu rama local. Esto es útil para mantener tu rama local actualizada con los cambios realizados por otros desarrolladores en el repositorio remoto. 

git tag
Este comando se utiliza para listar las etiquetas existentes en el repositorio de Git. Al ejecutarlo, Git mostrará una lista de todas las etiquetas que has creado en tu repositorio, lo que te permite ver fácilmente los puntos específicos en el historial del repositorio que has marcado con etiquetas. 

git tag -a nombreEtiqueta -m "Mensaje de la etiqueta"
Este comando se utiliza para crear una nueva etiqueta en Git. Al ejecutarlo, Git asignará la etiqueta especificada al commit actual en el historial del repositorio. Las etiquetas son útiles para marcar puntos específicos en el historial del repositorio, como versiones releases. 

git push origin nombreEtiqueta
Este comando se utiliza para enviar una etiqueta específica al repositorio remoto. Al ejecutarlo, Git subirá la etiqueta que has creado a tu repositorio remoto, lo que permitirá que otros desarrolladores vean y utilicen esa etiqueta en el historial del repositorio.

Branches o Ramas

git branch
Este comando se utiliza para visualizar las ramas creadas en nuestro repositorio local. También nos marca en un color la rama en la que estamos trabajando. 

git branch nombreRama
Este comando se utiliza para crear una nueva rama (si no existe aún) en nuestro repositorio. 

git branch -m nombreRama nuevoNombre
Este comando se utiliza para cambiar el nombre de una rama por otro. 

git branch -d nombreRama
Este comando se utiliza para eliminar una rama existente en nuestro repositorio. 

git checkout rama
Este comando se utiliza para movernos entre las distintas ramas del repositorio, así como también si en lugar de indicarle una rama, se le indica una versión, esto nos permitirá visualizar los archivos y sus contenidos según la versión especificada. 

git checkout -b nombreRama
Este comando se utiliza para crear una nueva rama (si aún no existe en nuestro repositorio) y movernos hacia ella con un solo comando
